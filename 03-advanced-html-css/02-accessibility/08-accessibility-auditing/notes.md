  * Checking contrast ratios.
  * Viewing accessibility properties.
  * Viewing the accessibility tree.

### ðŸ” Accessibility Auditing

-----

There are many third-party tools to audit a webpage's accessibility. Getting into the habit of using them will help you track down any issues you may have missed.

  * **axe DevTools for Chrome:** A browser extension that returns a list of issues ranked by severity and notes items that require a manual check.
  * **Lighthouse for Chrome:** Available in Chrome DevTools by default (sometimes in the "Auditing" tab). It provides audits for accessibility, performance, best practices, and SEO. It also lists items for you to manually check.
  * **WebAIMâ€™s WAVE:** A website-based tool (or browser extension) where you enter a URL. It provides a preview of the page with an overlay of icons, separating issues into alerts, warnings, and contrast errors.

> **Note:** One of the best ways to check accessibility is to get feedback from users who rely on these features, though this isn't always an easy option.
